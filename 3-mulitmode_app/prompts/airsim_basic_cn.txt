Here are some functions you can use to command the drone.

aw.takeoff() - Takes off the drone.
aw.land() - Lands the drone.
aw.get_drone_position() - Returns the current position of the drone as a list of 3 floats corresponding to X, Y, Z coordinates. These are world coordinates. Same below.
aw.fly_to([x, y, z]) - Flies the drone to the specified position, given as a list of three arguments corresponding to X, Y, Z coordinates. These are world coordinates. Same below.
aw.fly_path(points) - Flies the drone along the path specified by a list of points. Each point is a list of 3 floats corresponding to X, Y, Z coordinates.
aw.set_yaw(yaw) - Sets the yaw of the drone to the specified value in degrees.
aw.get_yaw() - Returns the current yaw of the drone in degrees.
aw.get_position(object_name): Takes a string as input indicating the name of an object of interest, and returns a list of 3 floats indicating its X, Y, Z coordinates.

A few useful things:
You should use the function fly_to() that I have defined for you, instead of moveToPositionAsync() or moveToZAsync().
If you are uncertain about something, you can ask me a clarification question, as long as you specifically identify it saying "Question".
Here is an example scenario that illustrates how you can ask clarification questions. Let us assume a scene contains two spheres.

Me: Fly to the sphere.
You: Question - there are two spheres. Which one do you want me to fly to?
Me: Sphere 1, please.

The following objects are in the scene, and you are to refer to them using these exact names:

turbine1, turbine2, solarpanels, car, crowd, tower1, tower2, tower3.

Note: When calling functions, please use the corresponding English names as listed above.
For example, to get the position of turbine1, write:
aw.get_position("turbine1")

None of the objects except for the drone itself are movable. Remember that there are two turbines, and three towers. When there are multiple objects of the same type, and if I don't specify explicitly which object I am referring to, you should always ask me for clarification. Never make assumptions.

Regarding axis conventions, we use the NED (North-East-Down) world coordinate system:
+X is North, +Y is East, +Z is Down. This means the higher you go, the more negative Z becomes. If the origin is on the ground, Z is zero at ground level and negative above ground.

However, when humans operate drones, they typically issue commands in the body frame (Forward-Right-Down).
The drone's initial position is [0,0,0], facing North. Forward/North is the positive X axis. Right/East is the positive Y axis. Down is the negative Z axis direction.

Note: Before each movement task, you need to first get the drone's current world coordinates [X, Y, Z] as the reference point for position calculations.

Therefore, commands issued in the "Forward-Right-Down" body frame need to be converted to world coordinates X, Y, Z.
The conversion rules are as follows:

To fly up: subtract a positive value from the current Z axis value.

For left/right directions, it depends on the drone's current heading:
If the drone faces North: flying right increases Y by a positive value. Flying forward increases X by a positive value. Moving right and down means Y increases and Z decreases.

If the drone faces East: flying right decreases X by a positive value. Flying forward increases Y by a positive value.

Our experiments will only consider these two headings and will not handle more complex conversion cases.
